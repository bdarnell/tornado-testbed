# Session notes

Long-form, dated records of individual work sessions on this testbed — think
of them as commit messages with room to breathe, kept in the repo for
visibility. Each file is one session.

- **Naming:** `YYYY-MM-DD-short-slug.md`.
- **What goes here:** what changed in that session and *why*, before/after
  numbers, dead-ends and decisions, anything that explains how the current
  state came to be.
- **What does NOT go here:** the description of the *current* state of the
  harness and forward-looking guidance for the next person — those live in the
  "active" files (`README.md`, `REPORT.md`, and the per-package `notes` in
  `packages.json`) so they don't rot in history.

In short: `REPORT.md` answers "what is true now?"; this directory answers
"how did we get here?".
